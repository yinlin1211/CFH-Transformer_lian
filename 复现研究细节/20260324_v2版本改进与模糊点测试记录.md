# CFH-Transformer v2 版本改进与模糊点测试记录

**日期**：2026年3月24日
**作者**：Manus AI

## 1. 背景与目标

在对 CFH-Transformer 论文进行深度复现的过程中，我们发现原版复现代码（v1）在 Tokenization 模块的实现上直接沿用了 TRIAD [1] 的 Dilation（空洞卷积）方案。为了进一步对齐论文原文的描述，我们对论文文本和图表进行了极其严格的交叉比对，并在服务器上创建了 `CFH-Transformer_v2` 目录进行改进和测试。

本文档记录了 v2 版本的核心改进点、正在测试的模糊点，以及论文中尚未明确的其他细节。

## 2. v2 版本的核心改进（确定要改的）

通过对比论文原文，我们在 v1 和 v2 中已经落实了以下**有明确论文依据**的改进：

| 模块 | 改进内容 | 论文依据 |
|---|---|---|
| **Octave 维度** | `octave_depth` 从 3 改为 **4** | Section 3.3："kernels of size **(4**, 3/5/7, 3)" |
| **Time 维度** | `time_width` 从 1 改为 **3** | Section 3.3："kernels of size (4, 3/5/7, **3**)" |
| **位置编码** | 固定正弦 PE 改为 **可学习 PE (LearnablePE)** | Section 2.3 明确指出三个 Transformer 均使用 "learnable positional encoding" |
| **优化器** | AdamW 改为 **Adam** | Section 3.3："adopt the **Adam** optimizer" |
| **学习率调度** | 去除 10 epoch warmup，直接使用 **CosineAnnealingLR** | 论文全文未提及 warmup |
| **训练时长** | 从 300 epochs 提升至 **1300 epochs** | 由 Fig.4 X轴 160k iterations 推算得出 |
| **输出音高范围** | 88键钢琴范围改为 **48音 (C2~B5)** | 论文输出 $\hat{y} \in \mathbb{R}^{T \times N}$，N 为人声音高范围 |

## 3. 正在测试的核心模糊点：Pitch Class 卷积核设计

这是 v1 和 v2 之间**最核心的差异**，也是目前正在通过实验验证的假设。

### 争议点说明
- **v1 的实现（TRIAD 方案）**：`k_p=2`（固定），通过 `dilation=[0,28,16]` 跳跃采样来捕获基频、纯五度和大三度。
- **v2 的实现（连续大核方案）**：`k_p=[3,5,7]`，`dilation=1`，使用三个不同宽度的连续卷积核。

### 为什么 v2 改为连续大核？（证据链）
1. **文本证据**：论文 Section 3.3 明确写明 "kernels of **size** (4, 3/5/7, 3)"。在深度学习语境中，"size" 专指 kernel_size，不包含 dilation。且论文全文**未提及** dilation。
2. **视觉证据**：论文 Fig.3 画的三个卷积核是**实心连续矩形块**，宽度比例约 3:5:7。如果使用了 dilation 跳跃采样，图中应画成间隔的点阵。
3. **消融实验对比**：Section 3.6 强调自己的方案是 "**automatically** capture the harmonics"，而作为对比的 HPPNet [2] 是 "**pre-defined** harmonic distances"。TRIAD 的 dilation 本质上也是 pre-defined，这暗示 CFT 放弃了人为设定距离，转而使用连续大核让模型自己学。
4. **TRIAD 原文对比**：TRIAD [1] 原文明确指出其 pitch class 维度的 kernel size 固定为 2。CFT 论文写的是 3/5/7，这本身就说明 CFT 修改了 TRIAD 的 pitch class 维度设计。

**当前测试状态**：v2 正在服务器 GPU 0 上运行连续大核方案，后续将与 v1 的结果进行对比，以验证哪种方案更接近论文报告的 COnP=80.13%。

## 4. 论文中尚未明确的其他模糊点

除了上述正在测试的 Tokenization 方案外，论文中还有一些超参数完全没有给出具体数值，目前 v2 只能基于经验或保留原版设定：

| 参数 | v2 当前设定 | 模糊原因 |
|---|---|---|
| `num_cycles M` | **2** | 论文只说 "M refers to the number of times the cycle structure is performed"，未给具体值 |
| `conv_channels h` | **32** | 论文只说 "use h 3D harmonic convolution kernels to extract h channels"，未给具体值 |
| `h_dim H` | **128** | 论文未明确给出 H 的值 |
| Transformer `nhead` | **FH:8, HT:8, TF:6** | 论文未提及 |
| `dim_feedforward` | **512** | 论文未提及 |
| 学习率调度器 | **CosineAnnealingLR** | 论文只说 Adam lr=3e-4，完全未提及 scheduler |

## 5. 结论

v2 版本已经将所有有明确论文依据的细节对齐，并针对最核心的 Tokenization 模糊点提出了"连续大核"的假设并付诸测试。剩余的超参数模糊点可能需要在未来通过进一步的消融实验来确定最佳配置。
