# 类别不平衡问题研究报告

> 生成时间：2026年03月23日
> 作者：Manus AI

## 1. 问题背景

在自动音乐转录（AMT）和歌声转录（SVT）任务中，模型通常需要在帧级别（Frame-level）预测音符的起音（Onset）、持续（Frame）和尾音（Offset）。由于音乐的物理特性和时间分布，这一任务天然存在极端的**类别不平衡（Class Imbalance）**问题。

### 1.1 为什么 Onset 帧占比极少？

以一首标准的流行歌曲为例（假设采样率为 16kHz，hop_length 为 320，即每帧约 20ms）：
- 歌曲中包含大量的前奏、间奏和过渡段，这些区域没有任何人声音符，全部为静音帧（负样本）。
- 即便是有人声的段落，一个持续 1 秒的音符中，**Onset 帧通常只有 1 帧（占比 2%）**，而持续帧（Frame）有 49 帧（占比 98%）。
- 在整首 3~4 分钟的歌曲中（约 9000~12000 帧），Onset 帧的总数通常在 200~400 帧之间，**占总帧数的比例不到 5%**。

### 1.2 类别不平衡带来的后果

当使用标准的二元交叉熵（Binary Cross-Entropy, BCE）作为损失函数时，如果模型对所有帧都预测为 0（即"没有 Onset"），它在 95% 以上的帧上都能得到极低的 Loss。
这导致模型在训练时倾向于**保守预测**，输出的 Onset 概率普遍偏低。为了在推理时识别出这些音符，往往需要将二值化阈值大幅下调（如降至 0.05~0.20），这不仅增加了阈值搜索的成本，也容易引入更多的误报（False Positives）。

## 2. 领域内主流解决方案对比

针对这一经典问题，SVT/AMT 领域的不同研究提出了多种解决方案。以下是四种主流策略的对比分析：

### 2.1 策略一：加权 BCE 损失（Weighted BCE）

这是最直接、最容易实现的方案。通过给少数类（正样本）赋予更高的权重，强迫模型重视 Onset 的预测。

- **代表工作**：AV-SVT (NUS, 2023) [1]
- **实现方式**：在计算 BCE Loss 时，设置正样本权重 $w_{pos} = 15.0$。这意味着预测错一个 Onset 帧，其惩罚是预测错一个非 Onset 帧的 15 倍。
- **优点**：极低成本（只需修改一行代码），能有效促使模型输出更高的置信度，从而允许在推理时使用固定的 0.5 阈值。

### 2.2 策略二：标签平滑 + 三角窗（Label Smoothing with Triangular Window）

不将 Onset 严格限制在单帧的硬标签（1.0），而是将其"扩散"到相邻帧。

- **代表工作**：Phoneme-Informed SVT (KAIST, 2023) [2]
- **实现方式**：将原始的 `[..., 0, 0, 1, 0, 0, ...]` 标签平滑为 `[..., 0, 0.5, 1, 0.5, 0, ...]`。
- **优点**：增加了正样本的数量（从 1 帧变为 3~5 帧），同时赋予了模型对 Onset 时间微小偏移的容忍度，提升了鲁棒性。

### 2.3 策略三：Focal Loss

借鉴目标检测领域处理前景/背景极度不平衡的经验，使用 Focal Loss 动态调整样本权重。

- **代表工作**：High-Resolution Piano Transcription (Kong et al., 2021) [3]
- **实现方式**：使用公式 $FL(p) = -\alpha(1-p)^\gamma \log(p)$。当模型对大量负样本预测很有把握（概率接近 0）时，自动降低这些样本的 Loss 权重，将优化重心转移到难以区分的样本上。
- **优点**：无需手动硬编码正负样本比例，自适应能力强，在多项 AMT 任务中被证明有效。

### 2.4 策略四：完全绕开帧级预测（CTC 解码）

从根本上改变建模方式，不再进行逐帧的独立二分类预测。

- **代表工作**：CTC + Cross-Entropy (NTU, TASLP 2023) [4]
- **实现方式**：使用 CTC（Connectionist Temporal Classification）损失，模型直接预测音符序列。CTC 的动态规划解码机制不需要精确的帧级别对齐。
- **优点**：彻底消除了帧级别的类别不平衡问题，但需要对模型输出层和损失函数进行重大的架构调整。

## 3. CFH-Transformer 论文的处理方式分析

经过对 `CFH-Transformer` 原始论文的仔细研读，我们得出以下结论：

**论文完全没有提及类别不平衡问题，也没有采用上述任何一种针对性的解决方案。**

在论文 Section 3.3 (Implementation Details) 中，作者仅说明使用了标准的 Adam 优化器。这表明作者很可能使用了**均等权重的标准 BCE 损失**。

这种"不处理"的策略在早期或侧重架构创新的 SVT 论文中较为常见。作者主要依赖以下两点来变相缓解该问题：
1. **阈值搜索兜底**：通过在验证集上进行网格搜索，将阈值下调至极低水平（如代码中下探至 0.05），以补偿模型输出概率偏低的问题。
2. **强大的架构表征能力**：论文提出的 3D 谐波卷积结合多维度 Transformer，具备极强的特征提取能力，使得模型自身对 Onset 的判断置信度高于传统 CNN 模型，从而在均等权重下依然能达到 SOTA 性能。

## 4. 结论与建议

当前代码库中未处理类别不平衡（使用均等 BCE + 验证集阈值搜索），这**完全符合**论文的原始设计，并非复现遗漏。

然而，如果目标是进一步优化训练效率和模型泛化能力（超越原始论文），建议引入 **加权 BCE** 或 **Focal Loss**。这两种方法均只需在 `train.py` 中修改损失函数计算逻辑，无需改动核心网络架构，即可显著改善概率输出分布，减少对耗时阈值搜索的依赖。
