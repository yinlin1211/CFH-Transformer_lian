# CFH-Transformer v2 版本改进与模糊点测试记录

**日期**：2026年3月24日

---

## 1. 背景与目标

在对 CFH-Transformer 论文进行深度复现的过程中，我们发现原版复现代码（v1）在 Tokenization 模块的实现上直接沿用了 TRIAD [1] 的 Dilation（空洞卷积）方案。为了进一步对齐论文原文的描述，我们对论文文本和图表进行了极其严格的交叉比对，并在服务器上创建了 `CFH-Transformer_v2` 目录进行改进和测试。

本文档记录了 v2 版本的核心改进点、正在测试的模糊点，以及论文中尚未明确的其他细节。

---

## 2. 历史改动（v1 中已完成，v2 继承）

以下改动在 v1 中已经完成，v2 直接继承，不属于本次新增改动：

| 模块 | 改动内容 | 论文依据 |
|---|---|---|
| **Octave 维度** | `octave_depth` 从 3 改为 **4** | Section 3.3："kernels of size **(4**, 3/5/7, 3)" |
| **Time 维度** | `time_width` 从 1 改为 **3** | Section 3.3："kernels of size (4, 3/5/7, **3**)" |
| **位置编码** | 固定正弦 PE（`SinusoidalPE`）改为**可学习 PE（`LearnablePE`）** | Section 2.3 明确指出三个 Transformer 均使用 "learnable positional encoding" |
| **优化器** | AdamW 改为 **Adam** | Section 3.3："adopt the **Adam** optimizer" |
| **训练时长** | 从 300 epochs 提升至 **1300 epochs** | 由 Fig.4 X轴 160k iterations 推算得出 |
| **输出音高范围** | 88键钢琴范围改为 **48音 (C2~B5)** | 论文输出 $\hat{y} \in \mathbb{R}^{T \times N}$，N 为人声音高范围 |

> 注：固定正弦 PE 的版本保留在 GitHub 仓库的 `model_v2_backup.py` 中，可作为历史对照。

## 3. v2 相对 v1 的新增改动

v2 在 v1 基础上做了以下两处实质性改动：

| 模块 | v1 实现 | v2 实现 | 论文依据 |
|---|---|---|---|
| **Pitch Class 卷积核**（核心） | `k_p=2, dilation=[0,28,16]`（TRIAD 原版跳跃采样） | `k_p=[3,5,7], dilation=1`（连续大核） | Section 3.3 "kernels of **size** (4, 3/5/7, 3)"，全文未提 dilation |
| **学习率调度** | warmup(10 epochs) + CosineAnnealingLR | 直接 CosineAnnealingLR（无 warmup） | 论文全文未提及 warmup |

## 4. 正在测试的核心模糊点：Pitch Class 卷积核设计

这是 v1 和 v2 之间**最核心的差异**，也是目前正在通过实验验证的假设。

### 争议点说明

- **v1 的实现（TRIAD 方案）**：`k_p=2`（固定），通过 `dilation=[0,28,16]` 跳跃采样来捕获基频、纯五度和大三度。
- **v2 的实现（连续大核方案）**：`k_p=[3,5,7]`，`dilation=1`，使用三个不同宽度的连续卷积核。

### 为什么 v2 改为连续大核？（证据链）

以下 4 条独立证据支持连续大核方案：

**证据1（文本）**：论文 Section 3.3 明确写明 "kernels of **size** (4, 3/5/7, 3)"。在深度学习语境中，"size" 专指 kernel_size，不是 receptive field 或 effective size。且论文全文**未出现** dilation 一词来描述自己的方法。

**证据2（视觉）**：论文 Fig.3 画的三个卷积核是**实心连续矩形块**，宽度比例约 3:5:7。如果使用了 dilation 跳跃采样，应该画成间隔的点阵。

**证据3（消融实验）**：Section 3.6 强调自己的方案是 "**automatically** capture the harmonics"，而作为对比的 HPPNet [2] 是 "**pre-defined** harmonic distances"。TRIAD 的 dilation 本质上也是 pre-defined（人为设定纯五度/大三度），这暗示 CFT 放弃了人为设定距离，转而使用连续大核让模型自己学。

**证据4（与 TRIAD 对比）**：TRIAD [1] 原文明确指出其 pitch class 维度的 kernel size 固定为 $k_p=2$。CFT 论文写的是 3/5/7，这本身就说明 CFT 修改了 TRIAD 的 pitch class 维度设计。

**当前测试状态**：v2 正在服务器 GPU 0（Tesla V100-SXM2-32GB）上运行连续大核方案，后续将与 v1 的结果进行对比，以验证哪种方案更接近论文报告的 COnP=80.13%。

## 5. 论文中尚未明确的其他模糊点

除了上述正在测试的 Tokenization 方案外，论文中还有一些超参数完全没有给出具体数值，目前 v2 只能基于经验或保留原版设定：

| 参数 | v2 当前设定 | 模糊原因 |
|---|---|---|
| `num_cycles M` | **2** | 论文只说 "M refers to the number of times the cycle structure is performed"，未给具体值 |
| `conv_channels h` | **32** | 论文只说 "use h 3D harmonic convolution kernels to extract h channels"，未给具体值 |
| `h_dim H` | **128** | 论文未明确给出 H 的值 |
| Transformer `nhead` | **FH:8, HT:8, TF:6** | 论文未提及 |
| `dim_feedforward` | **512** | 论文未提及 |
| 学习率调度器 | **CosineAnnealingLR** | 论文只说 Adam lr=3e-4，完全未提及 scheduler 类型 |
